# Architecture Decision Records

[Back to Documentation Index](../README.md) | Previous: [Contributing Guide](../contributing.md)

Architecture Decision Records document important technical decisions and their context.

Use ADRs when a decision affects architecture, security, deployment, database design, or long-term maintainability.

## ADR Naming

Use this format:

```text
0001-short-decision-title.md
0002-use-keycloak-for-authentication.md
```

## ADR Template

```markdown
# ADR 0001: Decision Title

## Status

Accepted

## Context

What problem or decision needs to be addressed?

## Decision

What did we decide?

## Consequences

What are the benefits, tradeoffs, and risks?
```

## Suggested Initial ADRs

- Use Spring Boot for backend service.
- Use MySQL for persistence.
- Use Keycloak for authentication.
- Use simplified Git Flow for branch management.
