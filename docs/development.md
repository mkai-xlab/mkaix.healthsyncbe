# Development Guide

[Back to Documentation Index](README.md) | Previous: [Authentication Workflow](authentication.md) | Next: [API Documentation](api.md)

## Requirements

- JDK 21
- Maven 3.9+

## Local Run

Start the backend locally:

```bash
mvn spring-boot:run
```

The default local server port is `8080` unless overridden through Spring configuration.

## Build

```bash
mvn clean package
```

## Test

```bash
mvn test
```

## Configuration

Use `src/main/resources/application.yaml` for shared defaults.

Prefer Spring profiles for environment-specific values:

```text
application-dev.yaml
application-test.yaml
application-prod.yaml
```

Do not commit secrets. Use environment variables, deployment secrets, or local untracked configuration for sensitive values.

## Dependency Management

Dependencies are managed through `pom.xml` and the Spring Boot parent BOM.

Current key dependencies:

- `spring-boot-starter-webmvc`
- `spring-boot-devtools`
- `lombok`
- `spring-boot-starter-webmvc-test`

## Implementation Conventions

- Keep source code under the `com.g93.be` package so Spring component scanning discovers it automatically.
- Put HTTP endpoints in controller classes.
- Keep business logic in services instead of controllers.
- Keep persistence access behind repositories when a database is introduced.
- Use DTOs for request and response payloads when API contracts differ from domain objects.
- Add tests for new behavior when adding controllers, services, or persistence logic.

## Generated Files

The `target/` directory is generated by Maven. Do not edit or document behavior based only on generated files.
